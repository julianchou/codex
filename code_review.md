# 程式碼審查指引（Code Review）

審查目標：提升正確性、安全性、可維護性與一致性，同時保持最小變更。

## 審查清單
- 正確性：
  - 需求是否被滿足？是否有邏輯遺漏或邊界條件未處理？
  - 單元/整合測試是否涵蓋主要情境與錯誤情境？
- 安全性：
  - 是否引入潛在資安風險（注入、敏感資訊外洩、檔案權限）？
  - Secrets 是否以 placeholder 並由 CI/環境變數提供？
- 可讀性與一致性：
  - 命名、結構與格式是否一致？文件是否同步更新？
- 效能與可擴充：
  - 熱點路徑是否有不必要的複雜度或資源浪費？
- 風險與回滾：
  - 變更是否容易回滾？是否影響相依模組？

## 審查建議流程
1. 先讀 PR 描述與變更範圍，確認需求背景。
2. 以 `scripts/lint.ps1`、`scripts/test.ps1` 輔助驗證。
3. 針對具體行數留下可操作的建議與範例。
4. 若涉及安全/相依升級，確認 CI 的安全工作流結果（SARIF）。

## 合併前檢查
- [ ] PR 模板自我檢查清單皆通過。
- [ ] CI（lint/test/security）皆綠。
- [ ] 文件已更新（若適用）。
